# 💸 Predictive Fund Lapse & Allocation System

An end-to-end Machine Learning pipeline designed for Hackathons to predict government fund utilization, identify lapse risks, and recommend optimal budget allocations for the next financial year.

## 🚀 Features

### 1. 📊 Early Warning System (Lapse Prediction)
Uses a **Random Forest Classifier** to analyze spending velocity (H1 vs H2 spending) and historical performance to predict the **Probability (%)** that a department will fail to utilize its allocated budget before the end of the financial year.

### 2. 💰 Final Spend Forecasting
Uses a **Random Forest Regressor** to predict the *exact Rupee amount* a department will end up spending based on its current behavior, allowing us to estimate the exact amount of funds that will lapse.

### 3. 🎯 Smart Budget Reallocation (Zero-Based Budgeting)
Provides an AI-driven recommendation for next year's budget:
* **Efficient Departments** (Low Lapse Risk + Good History): Rewarded with **+10% growth** over their actual needed spend.
* **Inefficient Departments** (High Lapse Risk): Penalized with only a **+2% buffer** over their actual needed spend to force efficiency and reduce wasted allocation.
* **Standard Departments**: Given a standard **+5% growth** based on inflation.

### 4. 📈 Explainable AI
The output clearly states exactly *why* a department was flagged as high risk (e.g., "Very slow spending velocity in Q1+Q2" or "Historically poor utilization").

---

## 🛠️ Architecture

1. **`data_generator.py`**: A python script that simulates highly realistic JSON data for Departments, Vendors, and Transactions across the state of Maharashtra.
2. **`predictive_model.py`**: The core ML Engine. It ingests the JSON data (or connects to a real SQL database), performs feature engineering (calculating Utilizing Rates, Quarterly Spending percentages), trains two Random Forest models, and outputs a clean ASCII table of predictions.
3. **`processed_dashboard_data.csv`**: The clean, structured CSV file generated by the ML pipeline to be consumed by a frontend dashboard (e.g., Streamlit, React, PowerBI).

## 💻 Installation & Usage

### 1. Install Dependencies
Make sure you have Python 3 installed, then install the required AI and data manipulation libraries:
```bash
pip install pandas numpy scikit-learn tabulate sqlalchemy psycopg2-binary pymysql
```

### 2. Generate the Dataset
Run the data generator to create fresh, randomized JSON simulation files (`departments.json`, `vendors.json`, `transactions.json`):
```bash
python data_generator.py
```

### 3. Run the Predictive Model
Run the core ML pipeline. By default, it will detect the local JSON files and run the predictions:
```bash
python predictive_model.py
```

**(Optional) Connect to a Real Database**
You can seamlessly switch the model from JSON files to a live SQL Database (PostgreSQL, MySQL, etc.) by passing your connection string:
```bash
python predictive_model.py --db-uri="postgresql://user:password@localhost:5432/my_database"
```
